# AI LP制作プロンプト集

## 参考画像からLP要件を整理する

```text
あなたはLP制作ディレクターです。
添付した参考画像をもとに、同じ雰囲気のLPを制作するための要件を整理してください。

出力してほしい内容:
- ターゲット
- デザインコンセプト
- カラーパレット
- ファーストビュー構成
- 必要セクション
- CTA設計
- スマホ表示で重要な導線
- 実装時の注意点

ただし、丸パクリではなく、別案件として自然に使える構成にしてください。
```

## CodexにLPを実装させる

```text
あなたはReact / Vite / Vercel向けのフロントエンドエンジニアです。

以下の要件でLPを作ってください。

目的:
{業種}向けの問い合わせ・予約につながる1ページLP

デザイン:
{参考画像の雰囲気}

必須セクション:
- ヘッダー
- ファーストビュー
- 悩み訴求
- サービス内容
- 選ばれる理由
- 施術/サービスの流れ
- 料金
- 口コミ
- FAQ
- CTA
- フッター

実装条件:
- Vite + React
- PC/スマホ対応
- npm run build 成功
- 画像は public/images に配置
- CTAは目立つように
- テキストがはみ出さないように
```

## 修正依頼

```text
以下を修正してください。

- ファーストビューの画像がトリミングされすぎています
- スマホのCTAボタンが大きすぎます
- カード内の文字が読みづらいです
- 全体の高級感は残してください

修正後に npm run build を実行し、エラーがあれば直してください。
```

## Vercelエラー対応

```text
Vercelでビルドエラーが出ています。
ログを読み、原因を特定して修正してください。

確認してほしい点:
- Build Command
- Output Directory
- base path
- 画像パス
- npm install の依存関係
- package.json の scripts

修正後、GitHubにpushできる状態にしてください。
```

## ポートフォリオ掲載

```text
この制作実績をポートフォリオに追加してください。

必要な作業:
- liveUrlを確認
- Playwrightでファーストビューのスクリーンショットを取得
- public/assets内の既存構成に合わせて画像保存
- 制作実績カードに追加
- npm run build
- GitHubへpush

カード説明は「何を作ったか」だけではなく「どの導線を改善したか」も伝えてください。
```
